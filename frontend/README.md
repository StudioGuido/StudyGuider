# How to get started

1. cd frontend
2. npm i
3. npm run dev


# File structure
```
frontend/
├─ eslint.config.js          # Lint rules (style & best practices for JS/React)
├─ index.html                # Single HTML shell; React mounts into <div id="root">
├─ package.json              # Project manifest: dependencies + npm scripts
├─ package-lock.json         # Exact versions lockfile (auto-generated by npm)
├─ public/                   # Static files served as-is (e.g., /favicon.svg, PDFs)
├─ README.md                 # Project notes, setup steps, decisions
├─ vite.config.js            # Vite dev/build configuration (aliases, plugins, env)
└─ src/                      # All application source code (bundled by Vite)
   ├─ App.css                # Styles scoped to App (optional if using Tailwind)
   ├─ App.jsx                # Root component; defines routes/layout composition
   ├─ assets/                # Imported assets that get optimized by Vite
   │  └─ (images, svgs, etc.)
   ├─ index.css              # Global styles (resets, Tailwind directives)
   ├─ main.jsx               # App entry point; renders <App /> into #root
   │
   ├─ components/            # Reusable, presentational building blocks
   │  ├─ Layout.jsx          # App chrome wrapper (navbar + page container)
   │  ├─ Navbar.jsx          # Top navigation / brand / user menu
   │  ├─ Tabs.jsx            # Small UI for switching modes (e.g., Summary↔AskAI)
   │  ├─ Button.jsx          # Shared button styles/behavior
   │  ├─ Loader.jsx          # Loading spinner/skeleton component
   │  └─ ErrorBox.jsx        # Consistent error display
   │
   ├─ pages/                 # Route-level screens (one folder per feature/page)
   │  ├─ Books/
   │  │  └─ components/
   │  │     └─ TextbookCard.jsx        # Presentational card for a single textbook (no fetching)
   │  │  └─ Books.jsx        # “My Books” list (links into Chapters)
   │  ├─ Chapters/
   │  │  └─ Chapters.jsx     # Chapters list for a selected book
   │  ├─ Understanding/
   │  │  └─ Understanding.jsx# PDF viewer + Summary/AskAI split view
   │  └─ Reinforce/
   │     └─ Reinforce.jsx    # Flashcards/Quiz + redirect to Results
   │
   ├─ services/              # Data layer (fake API now; real API later)
   │  ├─ fakeApi.js          # Mocked endpoints returning local data (with delay)
   │  └─ api.js              # Real HTTP calls (keep same function names to swap)
   │
   ├─ context/               # App-wide state providers (Auth, UI, Theme)
   │  └─ AuthContext.jsx     # Signed-in user info available app-wide
   │
   ├─ hooks/                 # Custom React hooks (encapsulate logic)
   │  └─ useAuth.js          # Helper to consume AuthContext safely
   │
   ├─ data/                  # Local JSON/JS mocks for development
   │  └─ mockData.js         # users, books, chapters, summaries, flashcards
```




# React + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Babel](https://babeljs.io/) (or [oxc](https://oxc.rs) when used in [rolldown-vite](https://vite.dev/guide/rolldown)) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).

## Expanding the ESLint configuration

If you are developing a production application, we recommend using TypeScript with type-aware lint rules enabled. Check out the [TS template](https://github.com/vitejs/vite/tree/main/packages/create-vite/template-react-ts) for information on how to integrate TypeScript and [`typescript-eslint`](https://typescript-eslint.io) in your project.
